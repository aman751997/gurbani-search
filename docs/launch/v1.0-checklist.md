## 1. Supabase — apply the cold-start timeout migration

**Why.** On the Supabase free tier the HNSW index evicts from memory after idle periods; the first query of the day can take ~5 s. The default `anon` role's 3 s statement_timeout aborts that query with a 503. Raising the `anon` timeout to 10 s fixes the cold-start blip without touching anything else.

**Steps.**

1. Open [Supabase Dashboard → your project → SQL Editor](https://supabase.com/dashboard).
2. Paste the contents of `supabase/migrations/0003_raise_anon_timeout.sql` and click **Run**.
3. Verify with:
   ```sql
   SELECT rolname, rolconfig FROM pg_roles WHERE rolname = 'anon';
   ```
   The `rolconfig` column should contain `statement_timeout=10s`.

## 8. **HARD GATE — manually review all 100 starter captions**

This is the R8 gate in the v1 plan. The semantic-search product promise is *"Finds your Gurbani. Never writes it."* — the 100 captions committed in `data/starter-captions.json` are the only LLM-written text that ships statically with v1.0. You must personally read each one and confirm none of them paraphrase scripture, quote more than a brief neutral phrase, or make authoritative interpretive claims.

**Steps.**

1. Open `docs/launch/caption-audit.md` — the audit script's report. It flags 13 captions as "fail" under a stricter 4-gram overlap heuristic. Many of these are false positives from formulaic 4-grams like `"the lord s name"` or `"the fear of death"` — these are background phrases in the BMS translation, not paraphrase. Read each flagged caption and the corresponding shabad row in `data/starter-captions.json`.
2. For each caption you want to drop or edit, there are two safe knobs:
   - **Edit in place** — open `data/starter-captions.json`, change the `explanation` field for the specific `shabad_id`. The static JSON is the source of truth for the homepage. The live `caption_cache` table in Supabase is separate — a future live query for the same `(query, shabad_id)` pair will still return the cached caption unless you also delete that row from `caption_cache`.
   - **Mark as guard** — set `explanation` to `null`, `confidence` to `"low"`, `source` to `"guard"`, and add `"guardTriggered": "manual-review"`. The homepage will render the neutral "No AI explanation" slot.
3. Do not paste new generated text. If a caption needs replacement, either hand-write something that is plainly about *the connection between query and shabad* (not the shabad's content), or mark the slot as guard-triggered.
4. When done, tick the box below.

- [ ] All 100 starter captions read and signed off.

   The "No AI explanation" slot will render on the next request.
